# v0: Describe a UI, Get Working Code

The first wave of AI coding tools focused on autocomplete — the model watching your keystrokes and suggesting the next line. Useful, but still fundamentally you writing code, token by token. v0 takes a different position in the workflow. You describe an interface in plain language, and it generates a complete, functional React component using real component primitives. Not a wireframe. Not a mockup. Code you can deploy.

The output is React built on **shadcn/ui** — a collection of beautifully designed, accessible components that have become the default component library for modern React development. A prompt like "a settings page with a sidebar nav, user profile form, and a danger zone for account deletion" produces working JSX with the actual shadcn components wired together, styled, and ready to drop into a project. The result isn't a simplified approximation; it uses the same `<Dialog>`, `<Form>`, `<Select>`, and `<Button>` primitives you would have reached for by hand.

v0 is also evolving beyond UI components into full-stack generation. The current platform can scaffold entire applications — not just the UI layer but database schemas, API routes, and deployment configuration. It connects to GitHub, deploys to Vercel directly, and has an agentic mode that breaks work into tasks and executes them. What started as a component generator has grown into something closer to an AI collaborator for the full development lifecycle.

## How It Works

The generation model is purpose-trained on React component patterns, shadcn/ui, Tailwind CSS, and the conventions of modern Next.js development. When you submit a prompt, the model doesn't just produce generic HTML — it reasons about layout, interaction patterns, accessibility, and component composition. A form component will include proper label associations. A data table will use the correct shadcn `<Table>` primitives with sorting and filtering wired up, not a raw HTML table.

The interface runs a **live preview** alongside the code. As the model generates, you can see the component render in real time. If the result isn't quite right, you iterate with follow-up prompts: "make the sidebar collapsible," "add a search input above the table," "use a sheet instead of a dialog for the edit form." Each iteration updates the code and the preview together.

**Visual edit mode** provides fine-grained control without prompting. You can click directly on elements in the preview and adjust properties — colors, spacing, typography — with the code updating to match. This bridges the gap between the declarative power of prompt-driven generation and the precise control of direct manipulation.

For larger applications, v0 supports a **project mode** with an agentic architecture. You describe the full application — what it does, what pages it has, what data it manages — and the system breaks the work into tasks, executes them in sequence, and assembles a coherent application. It can connect to a **Neon database** for persistence, wire up authentication, and configure the Vercel deployment. The output isn't just a prototype; it can be a running application.

**GitHub sync** means the generated code isn't trapped in a browser. v0 can push code to a repository, opening pull requests for review or committing to a branch for continued development in your normal editor. Once in a repository, the code is yours — no vendor lock-in, no proprietary format.

Deployment is one click. Vercel is the natural host, and the integration is direct. A generated application can go from prompt to live URL in minutes, including SSL, CDN distribution, and serverless function routing — all the platform infrastructure Vercel provides.

## In the Wild

- **Prototyping for design reviews**: Generate a working UI from a spec or Figma description in minutes, share a live URL with stakeholders instead of static screenshots.
- **Scaffolding standard CRUD interfaces**: Admin panels, settings pages, data tables, and form-heavy UIs that follow predictable patterns but take time to build by hand.
- **Landing pages and marketing sites**: v0 handles the kind of polished, component-rich marketing pages that typically require a designer and several hours of implementation.
- **Learning by example**: Developers newer to shadcn/ui or React can prompt for an example of a pattern they want to understand and read real, idiomatic code.
- **Design system exploration**: Generate multiple variations of a component — different layouts, different interaction models — to evaluate options before committing to an implementation.

## What It Doesn't Do

v0 generates React code; it doesn't generate Angular, Vue, or framework-agnostic HTML. The output is built around shadcn/ui and Tailwind — excellent defaults, but teams with established component libraries or alternative styling systems will need to adapt the output. Complex business logic with intricate state machines or deeply custom behavior will need human review and refinement; the model excels at structural and visual patterns, not domain-specific rules. And while the agentic project mode is powerful, it's still AI generation — architectural decisions in generated applications should be reviewed before production use, not accepted wholesale.
